## 6.6 传感器看门狗设计（EMERGENCY 冗余）

### 问题背景

EMERGENCY 停车机制依赖局部感知（激光雷达/深度相机），若传感器故障导致局部避障失效，EMERGENCY 无法触发。

### 解决方案：传感器心跳看门狗

架构：

```plain text
传感器节点 ──► sensor_watchdog ──► EMERGENCY 强制停车
(lidar/camera)    (独立进程)         (不依赖传感器数据)
```

设计要点：

```plain text
| 组件 | 说明 |
|------|------|
| `sensor_watchdog` | 独立进程，订阅各传感器心跳 Topic |
| 心跳频率 | 传感器应持续发布心跳（如 `/scan` 10Hz），超时 2s 无数据视为故障 |
| 故障响应 | 立即发布 `cmd_vel=0`，并通知 `fleet_coordinator` 进入 EMERGENCY 状态 |
| 恢复机制 | 传感器数据恢复后，需人工确认或延时自动恢复 |
```

ROS 2 实现：

```python
class SensorWatchdog(Node):
    def __init__(self):
        super().__init__('sensor_watchdog')
        self.declare_parameter('timeout', 2.0)  # 超时阈值

        # 订阅传感器心跳
        self.create_subscription(LaserScan, '/scan', self.on_scan, 10)
        self.create_subscription(Image, '/camera/depth', self.on_depth, 10)

        self.last_scan_time = None
        self.last_depth_time = None
        self.emergency_pub = self.create_publisher(Twist, '/cmd_vel_emergency', 10)
        self.timer = self.create_timer(0.1, self.check_timeout)

    def on_scan(self, msg):
        self.last_scan_time = self.get_clock().now()

    def check_timeout(self):
        now = self.get_clock().now()
        if self.last_scan_time and (now - self.last_scan_time).seconds() > 2.0:
            # 传感器故障，强制 EMERGENCY
            cmd = Twist()  # zero velocity
            self.emergency_pub.publish(cmd)
            self.get_logger().warn('Lidar timeout - EMERGENCY STOP')
```

优先级：高于 fleet_coordinator 的速度缩放 — 看门狗的 cmd_vel_emergency 直接到底盘，优先级最高。
